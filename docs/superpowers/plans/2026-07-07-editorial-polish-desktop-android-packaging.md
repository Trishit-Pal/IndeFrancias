# Editorial Polish + Desktop/Android Packaging (M2.5) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Apply an editorial-French-chic visual pass across FrenchPath's existing routes, fill the one missing UX surface (a real lesson-complete moment), and package the same static build as a Windows `.exe` (Tauri v2) and a sideload Android `.apk` (Capacitor) — all offline, all on-device, zero new servers.

**Architecture:** One SvelteKit `adapter-static` build (`frenchpath/build/`) stays the single source of truth. Web (Vercel) is untouched. A new `src-tauri/` wraps the build in WebView2 for the `.exe`. The existing (but never-initialized) Capacitor Android platform is added and finished for the `.apk`. Visual work extends existing token files (`src/lib/theme/`) rather than introducing a parallel design system.

**Tech Stack:** SvelteKit 2 / Svelte 5 (runes), Tailwind v4, Capacitor 8 (Android), Tauri v2 (new), Vitest, Playwright.

**Spec:** [docs/superpowers/specs/2026-07-07-editorial-polish-desktop-android-packaging-design.md](../specs/2026-07-07-editorial-polish-desktop-android-packaging-design.md)

## Global Constraints

- All commands run with `frenchpath/` as the working directory unless stated otherwise.
- CSP stays `default-src 'self'` everywhere — web, Tauri, and Capacitor shells. No exceptions, no CDN assets.
- No new runtime dependencies for the app bundle. `@tauri-apps/cli` is a **devDependency only** (build tool, not shipped code).
- No Google Play Store / AAB work this phase — Android output is a sideload-signed `.apk` only.
- Keystore files and signing passwords are **never committed** — env vars only, keystore paths added to `.gitignore`.
- Every new CSS animation must be inert under `prefers-reduced-motion: reduce` (existing pattern: media query in `src/lib/theme/animations.css` + the `.reduce-motion` class toggled in `src/routes/+layout.svelte:79`).
- No new component-testing framework — this codebase tests logic (`*.spec.ts` under `src/lib/**`) with Vitest and flows with Playwright (`frenchpath/e2e/`); it does not unit-test Svelte component markup. Follow that convention; don't introduce `@testing-library/svelte` or similar.
- Existing suites must stay green after every task: `npm run check`, `npm run lint`, `npm run test:unit -- --run`, `npm run test:e2e`, `npm run spec:validate`.
- Reuse existing tokens/utilities before adding new ones (`src/lib/theme/tokens.css`, `src/lib/theme/animations.css`, the `fp-stamp` keyframe already used in `AchievementToast.svelte` and `layout.css:542`).

```typescript
/** True only inside the Tauri desktop shell (Windows/.exe WebView2). */
export function isDesktopPlatform(): boolean {
	if (typeof window === 'undefined') return false;
	return '__TAURI_INTERNALS__' in window;
}
```

Tauri v2 always injects `window.__TAURI_INTERNALS__` for its own IPC, independent of the `withGlobalTauri` config option (which only controls the separate, unused `window.__TAURI__` convenience API) — this makes detection reliable without opting into that broader global API surface.
