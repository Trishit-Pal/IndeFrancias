# M2.6 — Complete Local Product (design)

Date: 2026-07-10 · Status: approved (user, 2026-07-10) · Precedes: implementation plan
Sequencing decision: **lands before the public launch** (user choice; the M3 native-speaker
proofread remains the hard launch gate and runs in parallel).

## Goal

Ship FrenchPath as a best-in-class, fully local product a user gets by **downloading one
file and installing it** on Windows (.exe) and Android (.apk), implementing every "Later"
item from SPEC §8 with the privacy constitution intact: speaking core (on-device ASR),
FSRS optimizer, writing rubric feedback, and E2EE device-to-device sync — plus opt-in
updates and final packaging.

## Decisions taken (with user)

| Decision | Choice |
|---|---|
| Scope | All four deferred enhancements + packaging |
| Sequencing | Before launch (M2.6; launch = M2 + M2.6 + M3 proofread) |
| ASR engine | Vosk WASM (`vosk-browser`), French small model (~41 MB), Worker-hosted |
| ASR model delivery | Bundled inside .apk/.exe; web PWA opt-in download, served from the app's own domain |
| Sync transport | Encrypted merge-file (no network, no relay, no local-network stack) |
| Updates | Opt-in version check (default OFF), static `version.json`, link-only, no auto-install |
| Invariant 4 | Amended: two user-consented, GET-only, off-by-default fetches (model + version.json), both served from the app's own domain. On web the model fetch is same-origin, so the web CSP stays literally `default-src 'self'`; only the native shells (whose CSP we author) reach out. High-severity spec edit — approved by user |
| Hosting | Model + `version.json` are static files on the app's Vercel domain (not GitHub Releases — release assets redirect via `objects.githubusercontent.com`, which breaks clean origin pinning). GitHub Releases hosts only the installer downloads (navigations, not fetches) |

Rejected: Web Speech API for pronunciation (sends audio to Google — violates invariants
1/2/4; supersedes the ml-roadmap V1 line), whisper.cpp (batch-only, too slow on budget
Android WebViews), native Android recognizer (nondeterministic, French packs unreliable),
zero-knowledge relay sync (introduces a backend), full auto-update (key management +
always-on phoning home).

## Build order

WP1 rubric → WP2 optimizer → WP3 speaking → WP4 sync → WP5 packaging. Cheapest-risk-first;
WP1/WP2 de-risk Worker/WASM plumbing for WP3; WP4 (touches data destruction) gets full
attention; WP5 bundles WP3's model.

## WP1 — Writing rubric feedback

- `src/lib/lesson/rubric.ts`: pure `(answer, exercise) → RubricHint[]`.
- Rules are JSON patterns in content packs; `content/schema.ts` gains optional
  `rubricRules: { id, match, hint (i18n key), severity: 'gentle' | 'correction' }[]`
  (zod-validated) so the native proofreader reviews them like content.
- Example: `/\bje suis (\d+)\b/` → "In French, age uses *avoir*: j'ai 25 ans."
- Wired into `text-answer` and `cloze` only. UI: inline note in the existing lesson
  feedback strip (ink on `--fp-jaipur-light`), `aria-live="polite"`, no new surface,
  no layout shift.

## WP2 — FSRS optimizer

- Dependency `fsrs-browser` (official open-spaced-repetition Rust→WASM optimizer).
- `src/lib/srs/optimizer.worker.ts`: reads `reviewLog` from IndexedDB, trains, posts
  back 21 weights. Runs ≤ once/week, only after a review session, never blocks UI.
- Keeps existing `optimizer.ts` exported interface (`FsrsOptimizerResult`,
  `FSRS_OPTIMIZER_MIN_REVIEWS = 500`); fills the stub body.
- Tuned weights + `optimizedAt` + review count stored in settings; `fsrs.ts` consumes
  via its existing params path. UI: one row on the progress/L'Atelier surface —
  "Personalized scheduling: on (tuned from N reviews)" + *Reset to defaults*.
- WASM served from `'self'` (bundled). If any WebView requires `wasm-unsafe-eval` in
  `script-src`, that is a separate high-severity spec edit raised at implementation time.

## WP3 — Speaking core

- **Engine:** Vosk WASM in a dedicated Worker; lazy model load on first speaking exercise.
- **Modules:**
  - `src/lib/speech/asr.ts` — Worker lifecycle, model load, transcribe API.
  - `src/lib/speech/score.ts` — pure scoring: Vosk word results vs expected phrase →
    per-word verdict (good / unclear / missed), normalized word-level Levenshtein;
    tested with JSON fixtures, no audio required.
  - `src/lib/speech/modelSource.ts` — resolves bundled asset (Capacitor/Tauri) vs
    Cache Storage (web, after opt-in download).
  - New exercise type `speak` (12th) + `SpeakExercise.svelte`; shadowing mode =
    audio + transcript scroll sync (pure UX, no ML).
- **Model delivery:** static asset inside .apk/.exe (installers ~55–90 MB). Web PWA:
  consent card "Download speaking pack (41 MB)" → same-origin fetch (model hosted on
  the app's own domain, excluded from the service-worker precache) → SHA-256 verified
  against a hash pinned in the app bundle → Cache Storage.
- **UI (product register, Le Grand Voyage):** per-word ink chips on paper tinted
  `--fp-sage` / `--fp-saffron` / `--fp-jaipur`; existing record-and-compare waveform
  retained above; tap a chip to hear that word via TTS. No gauges or percentage
  hero-metrics. Keyboard operable; `aria-live` summary; reduced-motion = no stagger.
- **Degradation:** WASM init failure (old WebView, low memory) falls back silently to
  record-and-compare + self-grade. Never a dead end.

## WP4 — E2EE merge-file sync

- **Crypto (WebCrypto only):** PBKDF2-SHA-256, 600 000 iterations, random 16-byte salt →
  AES-256-GCM, random 12-byte IV. No new dependencies. (Argon2 skipped: needs WASM;
  PBKDF2 at this count is adequate for an offline file. `ponytail` ceiling — swap to
  Argon2id if threat model changes.)
- Fills the existing `SyncEnvelope` (`src/lib/sync/types.ts`); `pushSyncEnvelope` /
  `pullSyncEnvelope` remain dead (file-only, no endpoint).
- `src/lib/sync/mergeFile.ts` — encrypt/decrypt around the existing backup payload.
- `src/lib/pwa/merge.ts` — per-record last-write-wins on per-store `updatedAt`
  (srsCards, reviewLog, activity, settings, skillProfile); monotonic counters
  (streak best, XP totals) merge by **max**, not LWW.
- **Safety:** decrypted payload runs the full validate-before-destroy pipeline
  (schema, checksum, version-range); dry-run merge computes a preview
  ("+214 reviews from your phone, 0 conflicts") shown in the same modal pattern as
  backup import before anything is written.
- **UI:** inside Settings → data-sovereignty panel: "Sync between your devices" —
  export/import encrypted sync file; passphrase field with strength hint; copy states
  the file is useless without the passphrase.

## WP5 — Packaging, distribution, opt-in updates

- `src/lib/platform/updates.ts`: Settings toggle (default **OFF**) → fetch static
  `version.json` from the app's own domain → semver compare → show a download link.
  Nothing auto-installs; toggle copy honestly notes the fetch reveals your IP to the host.
- Windows NSIS via existing `npm run build:desktop` (human step: VS Build Tools).
  Android via `docs/android-init.md` + release keystore (human step). Both embed the
  Vosk model. GitHub Releases + site /download page publish SHA-256 checksums per artifact.
- iOS remains deferred (no macOS available).

## Cross-cutting

**Error handling.** Every Worker has a terminal error path with a localized, friendly
message. ASR/optimizer failures degrade to existing behavior (self-grade / default
weights). Sync decrypt distinguishes wrong-passphrase from corrupt-file. All boundary
inputs validated: passphrase, imported files (schema+checksum), downloaded model bytes
(pinned SHA-256), version.json (zod).

**Testing (TDD per package; baseline 195 unit + 35 e2e stays green).**
- Unit: rubric rule matrix; scoring fixtures; merge matrix (both-changed, one-changed,
  monotonic counters, tamper rejection); crypto round-trip; semver compare.
- E2E: `speak` exercise with stubbed ASR worker; sync export→import round-trip;
  settings toggles; CSP suite extended to assert **no network calls** while toggles are off.

**Spec amendments (via /spec-sync, high-severity gate).**
- Invariant 4: exception clause — exactly two user-consented, opt-in, GET-only
  fetches (speaking-pack download; update check), both default-off, both served from
  the app's own domain; the web CSP remains literally `default-src 'self'`.
- §4: add CAP-SPEAK, CAP-UPDATE; CAP-SYNC planned→shipped; CAP-CONTENT gains rubricRules.
- §8: M2.6 "Complete Local Product" inserted; launch = M2 + M2.6 + M3 sign-off.
- ml-roadmap: V1 speech line corrected from Web Speech API to Vosk (privacy fix).

## Execution toolchain (user-mandated)

Implementation runs via **superpowers:subagent-driven-development** against the written
plan; every new UI surface is designed/reviewed under **impeccable** +
**frontend-design** (register: product; PRODUCT.md at repo root, written 2026-07-10);
**ponytail** governs minimalism. Each WP ends with code review, green gates
(`npm run check`, lint, unit, e2e), and `/spec-sync`.

## Non-goals

Local-network/QR sync (possible later on top of the merge engine), on-device writing
LLM (V2), whisper upgrade path, PathScene3D, iOS packaging, any store distribution,
auto-update, telemetry of any kind.
